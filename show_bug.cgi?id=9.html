Comment 2 Gary Holmlund 2012-01-04 07:39:45 EET 
I just experienced the same problem running a new installation of BibleTime 2.9.1 under arch linux. It turned out that the $HOME/.sword/sword.conf file was not present. When I manually added that file, the languages became available. To make the file manually put one blank line followed by the two following lines into the file.

[Install]
AugmentPath=/usr/share/sword

This file should automatically be created, but it seems that it is not created.

Comment 4 alvanx 2012-10-15 11:39:13 EEST 
I confirm on openSUSE 12.2. At first I thought this was a packaging problem (to where the book name translation files hadn't been packaged). Now I see that the problem might be with the installed directory. 

I have the sword.conf file in home/user/.sword and it points to /usr/share/sword/ as it should. However, the directory /usr/share/sword/ does not exist on my system. 

It seems there is a problem with dependencies. openSUSE offers two versions of sword, one named "sword", one named "libsword-1_6_2" (both at 1.6.2+svn2679). The latter is installed as a dependency for bibletime, the former is not. Here is what these packages provide: 

 - libsword-1_6_2 installs only three files, 2 of which are documentation, the last goes to /usr/lib/libsword-1.6.2.so
 - sword installs locale files into /usr/share/sword and also installs all the converters into /usr/bin. 
 - bibletime installs into /usr/share/bibletime

Now, after installing "sword", I have the proper folder /usr/share/sword. And now I can choose my language of choice from the bibletime settings.

I think this is an OpenSUSE issue with packaging the Sword library, rather than an issue with BibleTime. It appears that on OpenSUSE 12.3 installing the "bibletime" package pulls in the "libsword_1_6_2-1.6.2+svn2679" package, but the latter does not include the Sword library locale files for book names etc.

As a workaround, please try if installing the "sword" package fixes the issue.
