By the looks of it, there fallback algorithm provided by Qt appears to be rather good. The problem at hand is caused by the way we have named the translation files.

If QLocale::system().name() returns "pt_PT" for your system, then BibleTime will search (in order) for the following files in the translations directory:

  bibletime_ui_pt_PT.qm
  bibletime_ui_pt_PT
  bibletime_ui_pt.qm
  bibletime_ui_pt
  bibletime_ui.qm
  bibletime_ui
  bibletime.qm
  bibletime

The bibletime_ui_pt_BR.qm file just doesn't happen to be on that list. But the file can easily and should probably be renamed. I guess we have similar problems with other translations.

The major problem, however, is not those few words but the approach to the text chosen. To give you a simple and quick example (of many)in english (on the applications menu's) we find the "File" word.
